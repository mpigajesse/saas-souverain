Architecture SaaS Souverain

Flux zero-knowledge  ·  Données souveraines sur cluster PME  ·  Réplication PostgreSQL distribuée

🔑 Hiérarchie des clés cryptographiques

DEK XChaCha20-Poly1305 · unique par PME
├▶ Chiffre données métier + journal CBOR
├▶ sealed_box X25519 → 1 blob / appareil autorisé
└▶ Argon2id → blob récup. sur relais (opaque)
 
Enrôlement nouveau nœud
1. Génère paire X25519 → QR code clé publique
2. Nœud autorisé → sealed_box(DEK, pub_key)
3. Nouveau nœud ouvre le blob → récupère DEK
4. Jeton usage unique invalidé

⚡ Règles de bascule (Failover)

≥ 3 nœuds — bascule automatique par quorum
Supervision Patroni · promotion standby sans intervention
2 nœuds — bascule manuelle uniquement
Commande ss-node failover · intervention PME
Fencing — jeton d'époque monotone
Ancien primaire bloqué · split-brain impossible
Alerte 3 → 2 nœuds — signalée au portail SaaS
PME informée : bascule manuelle seulement
Sinistre total — récupération PME
Blob relais + code Argon2id → DEK retrouvée
